Documentation - Filters

The filters configuration section is used to define message filters used by Harvester. A valid message filter must implement ICreateFilterExpressions.

<filters>
    <filter type="GreaterThanOrEqual" property="Level" value="Information" />
    <filter type="Equal" property="ProcessName" value="MyCustomProcessName" />
    <filter type="Not">
        <filter type="OrElse">
            <filter type="Contains" property="Message" value="MyContainsString" />
            <filter type="StartsWith" property="Message" value="MyStartswithString" />
        </filter>
    </filter>
</filters>

Note: All filters defined at the root level use an implicit AndAlso filter.

  • Comparison Filters
  • Composite Filters
  • String Filters

property

A SystemEvent property name. A property attribute is required for any comparison filter and will be ignored for any composite filter.
Default Value: none.

type

An assembly qualified name for a message filter type. All built in filter types allow for abbreviated names as documented below.
Default Value: none.

value

A SystemEvent property value. A value attribute is required for any comparison filter and will be ignored for any composite filter.
Default Value: none.

Comparison Filters

Equal

Based on the Equal expression. Creates a BinaryExpression that represents an equality comparison.

<filter type="Equal" property="" value="" />

Greater Than

Based on the GreaterThan expression. Creates a BinaryExpression that represents a "greater than" numeric comparison.

<filter type="GreaterThan" property="" value="" />

Greater Than Or Equal

Based on the GreaterThanOrEqual expression. Creates a BinaryExpression that represents a "greater than or equal" numeric comparison.

<filter type="GreaterThanOrEqual" property="" value="" />

Less Than

Based on the LessThan expression. Creates a BinaryExpression that represents a "less than" numeric comparison.

<filter type="LessThan" property="" value="" />

Less Than Or Equal

Based on the LessThanOrEqual expression. Creates a BinaryExpression that represents a "less than or equal" numeric comparison.

<filter type="LessThanOrEqual" property="" value="" />

Not Equal

Based on the NotEqual expression. Creates a BinaryExpression that represents an inequality comparison.

<filter type="NotEqual" property="" value="" />
Composite Filters

And Also

Based on the AndAlso expression. Creates a BinaryExpression that represents a conditional AND operation that evaluates the second operand only if the first operand evaluates to true.

<filter type="AndAlso">
    ...
</filter>

Not

Based on the Not expression. Creates a BinaryExpression that represents a conditional OR operation that evaluates the second operand only if the first operand evaluates to false.

<filter type="Not">
    ...
</filter>

Or Else

Based on the OrElse expression. Creates a BinaryExpression that represents a conditional AND operation that evaluates the second operand only if the first operand evaluates to true.

<filter type="OrElse">
    ...
</filter>
String Filters

All string filters are case-insensitive.

Contains

Based on Contains. Returns true if the value attribute occurs within this string, or if value is the empty string (""); otherwise, false.

<filter type="Contains" property="" value="" />

Ends With

Based on the EndsWith. Returns true if the value attribute occurs at the end of this string; otherwise, false.

<filter type="EndsWith" property="" value="" />

Starts With

Based on the StartsWith. Returns true if the value attribute occurs at the start of this string; otherwise, false.

<filter type="StartsWith" property="" value="" />
