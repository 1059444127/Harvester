Application Itegration

When using applications developed against the Microsoft .NET Framework 3.5 or earlier, all output will be captured by Harvester regardless of whether or not a debugger has been attached (i.e., launched via F5 or CTRL-F5). However, if developing an application with the Microsoft .NET Framework 4.0 or later, output will NOT be captured by Harvester when a debugger has been attached (i.e., launched with F5).

If developing an application with the Microsoft .NET Framework 4.0 or later consider using one of the following integration options to enable monitoring of debug output even if a native debugger is attached.

  • Log4Net - Harvester Appender
  • NLog - Harvester Target
  • Trace - TraceListener
